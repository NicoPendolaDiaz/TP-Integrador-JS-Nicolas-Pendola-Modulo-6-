Evaluación del módulo #6, #7 y #8
1.-Situación inicial
Misión será construir, iterar y demostrar el funcionamiento de una aplicación real utilizando Node.js y Express, aplicando conocimientos adquiridos en los  módulos  6, 7 y 8  del programa.
El proyecto deberá abordar, en tres etapas, los aspectos clave del desarrollo backend  moderno: 
- desde servir contenido web y gestionar rutas,
- hasta conectar con una base de datos,
- implementar un ORM, 
- proteger rutas con JWT y 
- exponer una API RESTful profesional.

2.- Nuestro objetivo
Desarrollar de manera individual una  aplicación web  funcional  con  Node.js  y Express  , que:
● Sirva contenido web dinámico.
● Gestione usuarios y datos persistidos en base de datos.
● Exponga una API RESTful con rutas protegidas mediante  JWT.
● Integre persistencia con archivos planos, ORM para manejo de relaciones y funcionalidades clave como subida de archivos.
La implementación será  progresiva  , en tres partes  alineadas a los módulos del curso:
→ Módulo 6: estructura inicial del servidor, rutas,  vistas y persistencia básica. Pág.  2
→ Módulo 7: integración con base de datos, modelos y operaciones CRUD usando ORM.
→ Módulo 8: diseño de una API RESTful segura, con subida de archivos y autenticación. 

3.- Requerimientos
Estos son los fundamentos que deberás respetar y aplicar durante todo el desarrollo del proyecto. Funcionan como condiciones base:
○ La aplicación deberá estar desarrollada en Node.js (v18  o  superior) utilizando el framework Express.js.
○ El proyecto debe estar estructurado en rutas, controladores, middlewares y servicios siguiendo una arquitectura modular.
○ Se espera el uso de Sequelize o Mongoose como ORM, según el tipo de base de datos seleccionada.
○ La persistencia debe implementarse sobre una base de datos relacional (PostgreSQL) o documental (MongoDB).
○ El repositorio del proyecto debe estar subido a GitHub, con estructura  clara y un archivo  README.md  que documente su uso e instalación.
○ Se deberá incluir al menos un mecanismo básico de seguridad utilizando JSON Web Tokens (JWT). 

3.1.- Requerimientos específicos
Estos son los  elementos concretos  que la aplicación  desarrollada deberá contemplar  funcional y técnicamente. Serán validados  en cada una de las entregas.
➜ Gestión general de la aplicación Pág.  3
● Servir contenido web estático o dinámico desde  Express.
● Implementar rutas públicas y privadas (protegidas  con JWT).
● Permitir autenticación de usuarios (login / registro).
● Aplicar lógica de persistencia en archivos planos para funcionalidades simples (logs, configuración, etc.).
● Permitir ejecución del servidor desde línea de comandos con nodemon o scripts.
➜ Acceso y gestión de datos
● Conectar la aplicación a una base de datos real (MongoDB  o PostgreSQL).
● Modelar y relacionar entidades utilizando un ORM (relaciones  1:1, 1:N y N:M).
● Permitir operaciones CRUD completas sobre al menos dos entidades clave.
● Implementar consultas filtradas y búsquedas dinámicas.
● Aplicar manejo de errores y validaciones en las operaciones con base de datos.
➜ Seguridad y API
● Crear una API RESTful con rutas protegidas mediante autenticación JWT.
● Implementar subida de archivos (por ejemplo, imágenes de usuario).
● Validar tipos de archivo y tamaño  en el endpoint de  subida.
● Estructurar las respuestas de la API con formato consistente (status, message, data).

Parte 1 – Módulo 6:
Primeros pasos con Node y Express 
Este proyecto será avanzado paso a paso en cada clase y podrás completarlo progresivamente a través de los espacios asincrónicos. También tendrás espacios de consulta en las clases sincrónicas para despejar tus dudas. Todo gran desarrollo backend comienza con una base sólida. En esta primera parte del desafío, deberás levantar los  cimientos de tu aplicación. Como responsable técnico del proyecto, tu rol será:
- iniciar el servidor con Node.js y Express,
- configurar la estructura de carpetas,
- implementar las primeras rutas y servir contenido web.
Esta etapa es clave: lo que construyas aquí se convertirá en la base del sistema que más adelante integrará una base de datos y expondrá una API segura. Por eso,  además, de que el servidor funcione, tu objetivo será dejar un entorno bien organizado, claro y modular.

● Objetivo: 
→ Comprender el entorno Node.js y el rol de Express en el desarrollo backend moderno.
●  Tareas:
→ Investigar el ecosistema Node.js y para qué se utiliza.
→ Identificar qué aporta Express sobre Node puro.
→ Realizar un  esquema visual del flujo  básico servidor–cliente.
● Tarea PLUS (opcional):

4.2.-  Instalación y configuración de Node  (Lección 2)
● Objetivo: Preparar el entorno de desarrollo y configurar el proyecto base.
● Tareas:
→ Instalar Node.js correctamente (mínimo versión 18).
→ Inicializar el proyecto con npm init y completar todos los campos relevantes del package.json. Pág. 5
→ Crear el archivo principal index.js o app.js, con una función que imprima "Servidor iniciado".
● Criterio de cumplimiento:
→ El proyecto debe ejecutarse sin errores desde terminal usando node index.js.
→ Se debe justificar brevemente en el README por qué se eligió index.js o app.js como nombre base del archivo principal.
● Tarea PLUS:
→ Configurar uso de variables de entorno (dotenv) para el puerto del servidor.
4.3.- Gestión de paquetes en Node (Lección 3)
● Objetivo: Incorporar y configurar paquetes clave para el desarrollo backend.
● Tareas:
→ Instalar y declarar las siguientes dependencias:
○ express (requerido)
○ dotenv (recomendado)
○ nodemon como devDependency
→ Crear scripts personalizados en package.json:
○ npm start
○ npm run dev
→ Agregar comentarios en el código para describir qué hace cada línea relevante.
● Criterio de cumplimiento:
→ Los scripts deben ejecutarse correctamente.
→ El  README  debe  incluir  instrucciones  claras  de  instalación  y ejecución.
→ Justificar por qué se eligieron esos scripts (si  se  cambió  el nombre, explicar la decisión). Pág.  6
4.4.- Sirviendo contenido web (Lección 4)
● Objetivo: Crear las primeras rutas del servidor y servir contenido al usuario.
● Tareas:
→ Crear al menos 2 rutas públicas (/ y /status), cada una con respuestas en HTML o JSON.
→ Configurar la carpeta /public para servir al menos 1 archivo estático.
→ Agregar middleware  express.static() correctamente.
● Criterio de cumplimiento:
→ La aplicación debe servir contenido estático desde  /public .
→ Al  menos  una  ruta  debe  devolver  respuesta  en  JSON y otra en HTML.
→ Justificar en el README si se decide no utilizar /public, explicando la alternativa elegida (por  ejemplo,  motor  de plantillas).
● Tarea PLUS (opcional):
→ Implementar una vista dinámic a con ejs u otro motor de plantillas.
4.5.- Persistencia en archivos planos (Lección 5)
● Objetivo: Implementar almacenamiento simple de datos usando el sistema de archivos.
● Tareas:
→ Crear un archivo log.txt y una función que registre en él cada visita a una ruta específica.
→ Usar fs.appendFile() para agregar líneas de texto.
→ Validar que el texto registrado tenga la siguiente estructura mínima: fecha, hora, ruta accedida.
● Criterio de cumplimiento:
→ El archivo debe registrar al menos 3 accesos simulados . Pág.  7
→ Justificar si se elige otro tipo de evento para registrar (por ejemplo, errores o inicio del servidor).
4.6.- Ejecución de un aplicativo Node (Lección 6)
● Objetivo: Validar que el servidor funcione correctamente y documentar la estructura del proyecto.
● Tareas:
→ Ejecutar la aplicación con npm run dev.
→ Validar el acceso a las rutas creadas.
→ Crear y completar el archivo README.md con:
○ Requisitos del sistema
○ Instrucciones de instalación
○ Ejemplos de uso
→ Subir el repositorio a GitHub con nombre claro y estructura organizada.
● Criterio de cumplimiento:
→ El repositorio debe incluir al menos 5 carpetas bien nombradas (routes, controllers, middlewares, public, logs).
→ Justificar en el README si se decide una estructura distinta (por ejemplo, incluir /utils, /services, etc.).
● Tarea PLUS (opcional):
→ Crear un router.js externo y conectar las rutas con app.use().
